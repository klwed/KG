# 知识图谱问答系统

## 项目概述

这是一个基于 **FastAPI** 的知识图谱智能问答系统（v2.0），集成了文档处理、知识图谱构建、RAG 检索增强问答、学生成绩管理等功能。

## 环境准备

### 1. 安装后端依赖

```bash
cd backend
pip install -r requirements.txt
```

### 2. 配置环境变量

复制 `.env.example` 为 `.env`，填入你的配置：

```bash
cp .env.example .env
```

编辑 `.env`:
- `DEEPSEEK_API_KEY`: DeepSeek API Key
- `NEO4J_PASSWORD`: Neo4j 数据库密码

### 3. 安装 Neo4j

下载并启动 Neo4j (社区版):
- https://neo4j.com/download/

启动后访问 http://localhost:7474

### 4. 安装前端依赖

```bash
cd frontend
npm install
```

### 5. 初始化数据库

```bash
cd backend
python -c "from core.database import init_db; init_db()"
```

## 使用流程

1. **注册登录**: 使用邀请码注册（教师自动注册第一个账号）
2. **上传文档**: 在"流程构建"页面上传 PDF/Word/TXT 文件
3. **向量化**: 将文档向量化存入 FAISS
4. **抽取三元组**: 从文档中提取知识三元组
5. **导入图谱**: 将三元组存入 Neo4j
6. **智能问答**: 在"图谱问答"页面提问，系统使用 RAG 检索回答

## 一、数据层

### 1.1 输入数据

- 离散数学 Word 文档
- 信息安全 Word 文档
- 学生掌握情况 Excel/CSV 表格

### 1.2 数据处理流水线

```
Word文档 → 文档解析 → 文本清洗 → 分块处理 → DeepSeek API 知识抽取 → 知识图谱 → Neo4j 图数据库
                                                                 ↓
                                              FAISS 向量化 ← 文档向量化
```

### 1.3 知识图谱节点与关系设计

**节点类型**：
- 课程：离散数学、信息安全等
- 章节：集合与关系、密码学基础等
- 知识点：等价关系、RSA算法等
- 知识标签：定义、对称性等描述性关键词
- 计算思维：分解、模式识别、抽象、算法思维、评估
- 学生：学生姓名
- 掌握情况：如"集合：优秀"、"函数：良好"等

**关系类型**：
- 包含：课程→章节，章节→知识点
- 体现：知识点→计算思维维度
- 描述：知识点→知识标签
- 掌握：学生→知识点
- 掌握程度：学生→掌握情况
- 属于：知识点→课程

### 1.4 掌握程度五级划分

| 级别 | 分数区间 | 说明 |
|------|----------|------|
| 优秀 | 90-100% | 完全掌握 |
| 良好 | 70-90% | 基本掌握 |
| 一般 | 50-70% | 部分掌握 |
| 较差 | 30-50% | 初步了解 |
| 很差 | 0-30% | 未掌握 |

### 1.5 知识点与计算思维映射

| 知识点 | 关联的计算思维 |
|--------|----------------|
| 集合 | 抽象、分解 |
| 函数 | 抽象、模式识别 |
| 关系 | 抽象、模式识别 |
| 图论 | 分解、模式识别、算法思维 |
| 命题逻辑 | 抽象、算法思维 |
| 谓词逻辑 | 抽象、算法思维 |
| 递归 | 分解、算法思维 |
| 排列组合 | 模式识别、算法思维 |

### 2.3 问答引擎（RAG 流程）

```
用户提问
    ↓
┌─────────────────────────────────┐
│  双路检索 (rag_engine.retrieve)  │
├─────────────────────────────────┤
│  1. 向量检索 (FAISS + text2vec) │
│  2. 图谱检索 (Neo4j)            │
│  3. 结果融合与排序              │
└─────────────────────────────────┘
    ↓
有结果 → 构建上下文 → DeepSeek 生成答案
    ↓
无结果 → 直接调用 DeepSeek 生成答案
```

## 四、核心模块详解

### 4.1 文档处理 (document_processor.py)

**功能**：解析和分块处理文档

```
支持格式：PDF、DOCX、DOC、TXT

处理流程：
1. parse_file() - 根据文件类型解析
2. clean_text() - 文本清洗（去除噪声）
3. chunk_text() - 按段落分块（默认1500字/块）
```

### 4.2 关系抽取 (relation_extractor.py)

**功能**：使用 LLM 从文本中抽取知识三元组

**抽取类型**：
- 层级结构：课程 → 章节 → 知识点
- 计算思维关联
- 知识标签描述

### 4.3 RAG 检索 (rag_engine.py)

**功能**：双路检索增强问答

```
dual_search() → fuse_results() → build_context()

向量检索：text2vec-base-chinese → FAISS
图谱检索：Neo4j 关键词/子图查询
结果融合：去重 + 排序
```

### 4.4 知识问答 (kg_qa.py)

**类**：
- `KGQuerier`：图谱查询器
- `KGQA`：问答应用

**问答流程**：
```
ask() → ask_with_rag() → RAG检索 → LLM生成
                              ↓
                    + 个人上下文（学生数据）
```

## 五、数据流图

### 5.1 知识图谱构建

```
上传文档
    ↓
document_processor.process_document()
    ↓
relation_extractor.extract_batch()
    ↓ (DeepSeek LLM)
三元组列表
    ↓
kg_importer.import_triples()
    ↓
Neo4j 图数据库
```

### 5.2 RAG 问答

```
用户问题
    ↓
ask_with_rag()
    ↓
rag_engine.retrieve()
    ├─ vector_store.search() → FAISS
    └─ kg_client.search() → Neo4j
    ↓
结果融合
    ↓
+ 个人学习上下文（可选）
    ↓
llm_answer() → DeepSeek
    ↓
答案 + 相关三元组 + 子图
```

### 5.3 学生数据分析

```
上传 CSV
    ↓
student_processor.load_from_csv()
    ↓
student_processor.generate_triples()
    ↓
├─ 学生 --[掌握]--> 知识点
├─ 知识点 --[属于]--> 课程
└─ 知识点 --[体现]--> 计算思维
    ↓
kg_importer.import_triples()
    ↓
Neo4j
```
