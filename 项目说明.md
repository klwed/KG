# 知识图谱问答系统

## 项目结构

```
kg-qa-system/
├── backend/                 # FastAPI 后端
│   ├── api/                # API 路由
│   ├── core/               # 核心配置
│   ├── schemas/            # Pydantic 模型
│   ├── services/          # 业务服务
│   │   ├── deepseek_service.py    # DeepSeek API
│   │   ├── neo4j_service.py       # Neo4j 图数据库
│   │   └── document_parser.py     # 文档解析
│   └── requirements.txt
├── frontend/               # Vue3 前端
│   ├── src/
│   │   ├── api/          # API 调用
│   │   ├── views/        # 页面组件
│   │   └── router/       # 路由配置
│   └── package.json
└── documents/             # 课程文档目录
```

## 环境准备

### 1. 安装后端依赖

```bash
cd backend
pip install -r requirements.txt
```

### 2. 配置环境变量

复制 `.env.example` 为 `.env`，填入你的配置：

```bash
cp .env.example .env
```

编辑 `.env`:
- `DEEPSEEK_API_KEY`: 你的 DeepSeek API Key
- `NEO4J_PASSWORD`: Neo4j 数据库密码

### 3. 安装 Neo4j

下载并启动 Neo4j (社区版):
- https://neo4j.com/download/

启动后访问 http://localhost:7474

### 4. 安装前端依赖

```bash
cd frontend
npm install
```

## 启动服务

### 启动后端

```bash
cd backend
uvicorn api.main:app --reload --host 0.0.0.0 --port 8000
```

### 启动前端

```bash
cd frontend
npm run dev
```

访问 http://localhost:3000

## 使用流程

1. **上传文档**: 在"文档管理"页面上传 PDF/Word/TXT 文件
2. **抽取三元组**: 点击"抽取三元组"从文档中提取知识
3. **查看图谱**: 在"知识图谱"页面可视化查看实体关系
4. **智能问答**: 在"智能问答"页面提问，系统优先从知识图谱检索回答

## 二、后端层

### 2.1 技术栈

- FastAPI（Python Web 框架）
- Neo4j（图数据库）
- FAISS（向量检索）
- BAAI/bge-m3（Embedding 模型）
- DeepSeek API（LLM）
- SQLite（用户数据）
- JWT（身份认证）

### 2.2 API 模块

#### /auth
- `POST /register` - 注册（选择角色：teacher/student）
- `POST /login` - 登录，返回 JWT Token

#### /graph
- `GET /overview` - 获取图谱概览
- `GET /student/{id}` - 获取某学生掌握情况子图
- `GET /query` - 按关键词检索相关子图节点

#### /qa
- `POST /ask` - 问答接口（RAG + LLM fallback）

#### /students（仅教师）
- `GET /list` - 获取所有学生列表
- `GET /{id}/graph` - 获取指定学生图谱
- `GET /overview` - 所有学生掌握情况汇总

### 2.3 问答引擎（RAG 流程）

```
用户提问 → 问题向量化(bge-m3) → 双路检索(Neo4j + FAISS) → 有结果→构建上下文→DeepSeek生成答案 → 返回答案+来源标注
                ↓                                              ↓
            无结果                                      直接调用DeepSeek
```

## 一、数据层

### 1.1 输入数据

- 离散数学 Word 文档
- 信息安全 Word 文档
- 学生掌握情况 Excel/CSV 表格

### 1.2 数据处理流水线

```
Word文档 → 文档解析(python-docx) → 按段落切分 → 文本分块 → DeepSeek API 知识抽取 → 知识图谱 → Neo4j 图数据库
```

### 1.3 知识图谱节点与关系设计

**节点类型**：
- 课程（课程）：离散数学、信息安全等
- 章节（章节）：集合与关系、密码学基础等
- 知识点（知识点）：等价关系、RSA算法等
- 知识标签（知识标签）：定义、对称性等描述性关键词
- 计算思维（计算思维）：分解、模式识别、抽象、算法思维、评估
- 学生（学生）：学生姓名
- 掌握情况（掌握情况）：如"集合：优秀"、"函数：良好"等

**关系类型**：
- 包含：课程→章节，章节→知识点
- 体现：知识点→计算思维维度
- 描述：知识点→知识标签
- 掌握：学生→知识点
- 掌握程度：学生→掌握情况
- 属于：知识点→课程

### 1.4 掌握程度五级划分

| 级别 | 分数区间 | 说明 |
|------|----------|------|
| 优秀 | 90-100% | 完全掌握 |
| 良好 | 70-90% | 基本掌握 |
| 一般 | 50-70% | 部分掌握 |
| 较差 | 30-50% | 初步了解 |
| 很差 | 0-30% | 未掌握 |

### 1.5 知识点与计算思维映射

| 知识点 | 关联的计算思维 |
|--------|----------------|
| 集合 | 抽象、分解 |
| 函数 | 抽象、模式识别 |
| 关系 | 抽象、模式识别 |
| 图论 | 分解、模式识别、算法思维 |
| 命题逻辑 | 抽象、算法思维 |
| 谓词逻辑 | 抽象、算法思维 |
| 递归 | 分解、算法思维 |
| 排列组合 | 模式识别、算法思维 |

### 1.6 图谱结构示意

```
                    离散数学
                        │
                    ┌───┴───┐
                    │       │
                  集合     关系
                    │       │
            ┌───────┴───────┐
            │               │
        抽象/分解      抽象/模式识别
            
            知识点 ──[体现]──> 计算思维
              │
         [描述]
              │
            知识标签

            学生 ──[掌握]──> 知识点
              │
         [掌握程度]
              │
           掌握情况
```
